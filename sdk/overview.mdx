# SDK Overview

The Erdo Python SDK provides a powerful framework for building intelligent AI agents. With simple, intuitive APIs, you can create sophisticated automation workflows that combine language models, web research, code execution, and custom integrations.


</CodeGroup>

## Quick Start

Here's a simple agent that automates email responses:

```python
from erdo import Agent, Step
from erdo.actions import llm
from erdo.tools import EmailTool

# Create your agent
agent = Agent(
    name="email_assistant",
    description="Automatically responds to emails with context-aware replies"
)

# Define the automation step
response_step = Step(
    agent=agent,
    key="generate_response",
    actiontype=llm.message(
        model="claude-sonnet-4",
        query="Generate a professional response to this email: {{email_content}}"
    )
)

# Add email integration
email_tool = EmailTool(
    smtp_host="smtp.gmail.com",
    credentials="{{email_credentials}}"
)
```

## Advanced Features

### Execution Modes

Control when and how steps execute:

```python
from erdo import ExecutionMode
from erdo._generated.condition import IsSuccess

# Iterate over data sets
iterate_step = Step(
    agent=agent,
    key="process_files",
    actiontype=codeexec.execute(code="process_file({{file}})"),
    execution_mode=ExecutionMode(
        mode="iterate_over",
        data="files",
        if_condition=IsSuccess()
    )
)
```

### Result Handlers

Process step outcomes and control workflow:

```python
from erdo import ResultHandler
from erdo._generated.condition import IsSuccess, IsError

success_handler = ResultHandler(
    step=response_step,
    type="final",
    if_conditions=IsSuccess()
)

error_handler = ResultHandler(
    step=response_step,
    type="retry",
    if_conditions=IsError(),
    max_retries=3
)
```

### Complex Workflows

Build sophisticated multi-step automation:

```python
# Research workflow
research_agent = Agent(name="research_assistant")

# Step 1: Search for information
search_step = Step(
    agent=research_agent,
    key="search",
    actiontype=websearch.search(query="{{research_topic}}")
)

# Step 2: Analyze findings
analysis_step = Step(
    agent=research_agent,
    key="analyze",
    actiontype=llm.message(
        model="claude-sonnet-4",
        query="Analyze these search results: {{steps.search.results}}"
    ),
    depends_on=[search_step]
)

# Step 3: Generate report
report_step = Step(
    agent=research_agent,
    key="report",
    actiontype=llm.message(
        model="gpt-4",
        query="Create a comprehensive report: {{steps.analyze.analysis}}"
    ),
    depends_on=[analysis_step]
)
```

## Best Practices

<Tabs>
  <Tab title="Agent Design">
    - **Single Responsibility**: Each agent should have a clear, focused purpose
    - **Descriptive Names**: Use clear, descriptive names for agents and steps
    - **Error Handling**: Always include error handling and retry logic
    - **Testing**: Test agents with various inputs before deployment
  </Tab>

{" "}
<Tab title="Performance">
  - **Caching**: Use memory actions to cache frequently accessed data -
  **Parallel Execution**: Use background execution for independent tasks -
  **Resource Management**: Monitor API usage and implement rate limiting -
  **Optimization**: Profile and optimize slow operations
</Tab>

  <Tab title="Security">
    - **Credential Management**: Store sensitive data securely
    - **Input Validation**: Validate all external inputs
    - **Access Control**: Implement proper permissions and visibility settings
    - **Audit Logging**: Log all agent actions for debugging and compliance
  </Tab>
</Tabs>
